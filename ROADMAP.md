# Ito Development Roadmap

**目标：** 把 ito 做成生产级别的 agent harness 框架

---

## Phase 1: 核心功能完善 (Week 1)

### 1.1 多 LLM Provider 支持
- [ ] Anthropic Claude provider
- [ ] 本地模型支持 (Ollama / LocalAI)
- [ ] 自定义 provider 接口文档
- [ ] Provider 统一抽象层

### 1.2 上下文管理
- [ ] Token 计数器
- [ ] 上下文窗口管理
- [ ] 自动截断策略
- [ ] 消息压缩 / 摘要

### 1.3 并行工具调用
- [ ] 并行执行多个工具
- [ ] 依赖管理
- [ ] 超时控制

### 1.4 错误处理
- [ ] 重试机制
- [ ] 优雅降级
- [ ] 错误恢复策略

---

## Phase 2: 执行引擎增强 (Week 2)

### 2.1 沙盒执行
- [ ] VM2 / Isolated-VM 集成

- [ ] 安全策略配置

### 2.2 状态持久化
- [ ] 会话保存 / 恢复
- [ ] 检查点机制
- [ ] 状态序列化

### 2.3 流式响应
- [ ] 流式输出支持
- [ ] 中间结果回调
- [ ] 实时进度反馈

## 优先级排序

### P0 - 必须有
- 多 provider 支持
- 上下文管理
- 错误处理
- 基础测试
- 文档

### P1 - 应该有
- 沙盒执行
- 状态持久化
- 调试工具
- CI/CD

### P2 - 可以有
- 性能优化
- 监控集成
- 社区建设

---

## 技术栈

- **语言：** TypeScript
- **验证：** Zod
- **测试：** Jest
- **构建：** tsc
- **文档：** TypeDoc + VitePress
- **CI：** GitHub Actions
- **沙盒：** isolated-vm / VM2
